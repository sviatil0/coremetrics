## Pillar E: live self-monitoring (SHIP NEXT)

The marketing claim "1.6 MB binary, ~60 MB resident" lives in the
README today. The actual app shows nothing about itself. Putting
its own RSS + CPU% on screen turns the claim into a continuously
verifiable demo. Every screenshot a user shares now also markets the
footprint.

Stages:

- E1. `SelfStats` namespace that exposes `selfRssBytes()` and
  `selfCpuPct()`. Backed by `SystemMetrics::topProcesses(N)` filtered
  by `getpid()`; fallback to platform-specific calls if necessary.
- E2. Footer chrome: add a `SelfStats::renderFooterBadge(Screen&)`
  helper that paints "self: RSS 60 MB | CPU 0.5%" at the right edge
  of the existing footer status row, just left of the EXIT button.
  Colored `Theme::accentGreen()` for the metric values, `Theme::textDim()`
  for the units, so it reads as a deliberate flex.
- E3. About tab: add a new "Self" section between Runtime and License
  showing RSS, peak RSS since launch, CPU%, binary size on disk,
  uptime within the process. Format identical to the existing Host
  block.
- E4. README: update the Lightweight pitch to mention "the app shows
  its own footprint live; toggle to the About tab to inspect".
- E5. CHANGELOG entry under v0.4.

## Pillar F: widget integration (Phase 2 of GUI spec)

v0.3 deferred actually wiring the 12 v0.3-era widget classes into the
UI. They exist + have unit tests but nothing renders them.

Stages, in priority order:

- F1. Gauge wired into the System tab as the CPU at-a-glance dial.
  Sits above the CPU card; replaces the giant "0.0%" number for
  CPU with a semicircular gauge whose needle tracks the percentage.
- F2. Tooltip on every truncated row cell (Processes tab) so long
  command lines stay discoverable on hover.
- F3. Modal confirm-quit: Q today is destructive; modal asks first.
- F4. StackedBar replaces the memory composition strip painted by
  `src/MemBreakdownStrip.cpp`. The widget is more flexible (named
  segments + tooltips) and the strip is the obvious first user.
- F5. Heatmap of per-core CPU history (N×T matrix) under the
  existing cores strip on the System tab.
- F6. TabStrip widget replaces the four hand-rolled `<button>` tab
  entries in base.xml. Removes the hand-tuned x-coord math currently
  in TabIndicator.cpp.
- F7. Toggle widget drives the SOUND ON/OFF chrome.
- F8. Slider widget drives the poll-interval CLI flag at runtime.
- F9. Dropdown widget for the `--top-sort` cpu/mem/io selection at
  runtime.
- F10. TreeView widget for the Processes tab tree mode.

F1-F3 are the highest visibility; F6-F10 are quality-of-life.

## Pillar G: refactor backbone (Phase 1.2 tail)

coremetrics.cpp is at 849 lines after slice 17. Remaining extract
targets:

- G1. Slice 18: LiveLoop. The main render + event loop body (~340
  lines) into `src/LiveLoop.cpp` behind `LiveLoop::run(Window*, ...)`.
- G2. Slice 19: small helpers (handleShutdownSignal, aboutTabActive,
  the two extern-only forward decl blocks) folded into a tiny

- G3. After slices 18 + 19, coremetrics.cpp should be under 100
  lines: includes + main() that calls ArgParse + HeadlessRunner +
  LiveLoop + Settings::save. Goal is a pure entry point.

## Pillar H: cross-platform polish

- H1. Linux: implement `CpuTemp::readCelsius()` (currently macOS
  + Windows only).
- H2. Windows: implement `SystemMetrics::readPerCoreCpu()` (currently
  stubbed; the per-core strip is empty on Windows).
- H3. Windows: implement `Battery::read()` IsPresent gate (the call
  works on a laptop but reports IsPresent=true on desktops too).
- H4. macOS arm64: fix the gatekeeper "unidentified developer" prompt
  on first launch from a Homebrew install. Signing already
  half-wired by the existing notarize workflow; finish the chain.

## Cadence

One PR per stage. Priority order:

1. **E1-E5** (self-monitoring) — finish in one cycle
2. **G1, G2** (slice 18, 19) — small + finishes the refactor pillar
3. **F1, F2, F3** (Gauge, Tooltip, Modal) — highest-visibility widget
   integration
4. **J1, J2, J3, J4** (docs refresh) — ensure README / DOCS reflect
   the new shape
5. **Tag v0.4.0** when E + G + F1-F3 + the docs refresh all merge

The remaining items (F4-F10, H, I) are nice-to-have and roll into
subsequent cycles without blocking the v0.4 cut.
